Forwarding Rules

Define how packets are forwarded between virtual nodes

✅ CAD-based Collision Avoidance
Channel Activity Detection (CAD) is performed automatically before every transmission. CAD checks whether the channel is clear before sending, which effectively prevents TX collisions. Because of this, the per-rule TX delay can safely remain at 0 ms (default).
The TX delay field is still available if you need a fixed offset for other purposes (e.g. coordinating timing between multiple repeaters on the same frequency). For normal operation, 0 ms is recommended.

⚠️ RX CRC Error Ratio per Channel 0 buckets
How to read: shows the fraction of received packets that failed CRC check, per minute. 0.0 = all packets valid (healthy RF). 1.0 = all packets corrupted (heavy interference or weak signal). Values consistently above ~0.3 indicate RF problems: interference, wrong SF/frequency settings, or signals below sensitivity threshold. Note: channel_e (SX1261) runs spectrum-scan mode and naturally produces many CRC errors from sub-threshold signals — this is expected, not a fault.
